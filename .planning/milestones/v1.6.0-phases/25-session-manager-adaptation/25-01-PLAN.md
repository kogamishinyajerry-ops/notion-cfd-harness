## Key differences from ParaViewWebManager

| Aspect | ParaViewWebManager (current) | TrameSessionManager (new) |
|--------|----------------------------|--------------------------|
| Container name | `pvweb-{session_id}` | `trame-{session_id}` |
| Entrypoint | `--entrypoint /entrypoint_wrapper.sh` | Default CMD (no override) |
| Command | `pvpython .../launcher.py /tmp/config.json` | `pvpython /trame_server.py --port {N}` |
| Config | JSON file at /tmp/launcher_config.json | No config file |
| Volume mounts | case_dir + adv_protocols.py + config JSON | case_dir only |
| Image verify | Checks `vtk.web.launcher` module | No image verification |
| Ready check | "Starting factory" in logs | HTTP 200 at http://localhost:{port} |
| Session URL | `ws://localhost:{port}/ws` | `http://localhost:{port}` |
</context>

1. **Class definition**: `class TrameSessionManager:` — mirrors `ParaviewWebManager` structure but uses trame-specific Docker commands.

2. **`__init__`**: Initialize `self._sessions: Dict[str, TrameSession]` (import TrameSession from api_server.models — create placeholder first, see Task 2), `self._port_allocator` cycling `PARAVIEW_WEB_PORT_RANGE_START..END`, `self._idle_check_interval = 60`, `self._idle_task: Optional[asyncio.Task] = None`. Import config values from `api_server.config` (PARAVIEW_WEB_IMAGE, PARAVIEW_WEB_PORT_RANGE_START, PARAVIEW_WEB_PORT_RANGE_END, PARAVIEW_WEB_IDLE_TIMEOUT_MINUTES).

3. **`_cycle_ports` / `_next_port`**: Identical to ParaviewWebManager — port range cycling.

4. **`validate_docker_available`**: Identical to ParaviewWebManager.

5. **`launch_session(session_id, case_dir, port=None, job_id=None) -> TrameSession`**:
   - Call `validate_docker_available()` — raise `TrameSessionError` if unavailable
   - Allocate port: `allocated_port = port if port is not None else self._next_port()`
   - Generate auth_key: `auth_key = secrets.token_urlsafe(16)`
   - Create session record early with `status="launching"`
   - Start container via `_start_container(session_id, allocated_port, case_path)` (no config JSON, no adv_protocols.py mount)
   - Poll `_wait_for_ready` with HTTP 200 check (trame serves HTTP, not WebSocket)
   - Update `status="ready"` and return session

6. **`_start_container(session_id, port, case_path) -> str`**:
   - Docker command: `docker run -d --platform linux/amd64 --name trame-{session_id} -v {case_path}:/data:ro -p {port}:9000 {PARAVIEW_WEB_IMAGE} pvpython /trame_server.py --port 9000`
   - NOTE: No `--entrypoint` override, no config JSON volume, no `adv_protocols.py` volume
   - Container name: `trame-{session_id}` (not `pvweb-{session_id}`)
   - Return container_id from stdout

7. **`_wait_for_ready(session_id, container_id) -> None`**:
   - Poll HTTP at `http://localhost:{port}` every 1s for up to 60s
   - Use `requests.get` or `aiohttp` to check for 200 response
   - If container exits during startup, raise `TrameSessionError` with logs
   - If timeout, raise `TrameSessionError`

8. **`shutdown_session(session_id)`**: Same pattern as ParaviewWebManager — set status="stopping", `docker kill trame-{session_id}`, remove from dict.

9. **`get_session(session_id)`**: Return session or None.

10. **`update_activity(session_id)`**: Update `last_activity = datetime.utcnow()`.

11. **`_idle_monitor` / `_shutdown_idle_sessions`**: Identical to ParaviewWebManager — check `now - session.last_activity > timedelta(minutes=PARAVIEW_WEB_IDLE_TIMEOUT_MINUTES)`, call `shutdown_session`.

12. **`start_idle_monitor` / `stop_idle_monitor`**: Identical to ParaviewWebManager pattern.

13. **`class TrameSessionError(Exception)`**: Custom exception (like `ParaViewWebError`).

14. **Singleton**: `_trame_session_manager: Optional[TrameSessionManager] = None` + `def get_trame_session_manager() -> TrameSessionManager:` — create on first call.

Note: Import `TrameSession` from `api_server.models` — if it does not exist yet, the import will fail at runtime. Create the model in Task 2 first. The class can reference `TrameSession` as a forward reference or you can use `from __future__ import annotations`.

IMPORTANT DOCKER COMMAND DETAIL:
- The container's internal port is ALWAYS 9000 (trame --port 9000 maps to host port {allocated_port})
- The volume mount is `-v {case_path}:/data:ro` (case directory mounted read-only)
- No other volume mounts, no config files, no entrypoint override
- Image: `PARAVIEW_WEB_IMAGE` (same image — now runs trame instead of ParaView Web)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> FastAPI | Untrusted input (job_id, case_dir) crosses here |
| FastAPI -> Docker daemon | API server spawns containers on behalf of clients |
| Docker container network | trame server inside container; host port exposed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-01 | Tampering | case_dir path traversal | mitigate | `_validate_case_dir` in visualization.py already enforces absolute path + allowed roots; pass-through unchanged |
| T-25-02 | Denial of Service | Port exhaustion (all ports in range used by long-lived sessions) | mitigate | Port range 8081-8090 (10 ports); idle timeout 30 min auto-reclaims; sessions also manually shutdown via DELETE endpoint |
| T-25-03 | Information Disclosure | Auth key exposed in URL query params or logs | accept | Auth key is randomly generated (secrets.token_urlsafe 16 bytes); not logged; only transmitted to authorized dashboard client |
| T-25-04 | Elevation of Privilege | Unvalidated case_dir launches container for arbitrary path | mitigate | `_validate_case_dir` already validates path is absolute, exists, and is under DATA_DIR/REPORTS_DIR; TrameSessionManager receives pre-validated path |
</threat_model>
