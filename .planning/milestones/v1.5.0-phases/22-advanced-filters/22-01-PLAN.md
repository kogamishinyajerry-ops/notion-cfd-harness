The class must extend `ParaViewWebProtocol` and implement 5 RPCs:

**`visualization.filters.clip.create`** (FILT-01.1):
- Params: `{insideOut: bool, scalarValue: float}`
- Creates `simple.Clip()` on the active source, set to scalar clip with the given threshold, inside-out as specified
- Extracts `filter.GetOutputPort()` proxy ID
- Calls `simple.Render()` + triggers `self._app.SMApplication.InvokeEvent("UpdateEvent",())` to push viewport
- Returns: `{filterId: int, proxyId: int, success: bool, error?: string}`

**`visualization.filters.contour.create`** (FILT-01.2):
- Params: `{isovalues: float[]}`
- Creates `simple.Contour()` on the active source, computes point data for isosurfaces, sets isovalues from params
- Extracts proxy ID
- Calls `simple.Render()` + `InvokeEvent("UpdateEvent",())`
- Returns: `{filterId: int, proxyId: int, success: bool, error?: string}`

**`visualization.filters.streamtracer.create`** (FILT-01.3):
- Params: `{integrationDirection: "FORWARD"|"BACKWARD", maxSteps: int}`
- Creates `simple.StreamTracer()` on the active source using velocity field U, with specified integration direction and max steps
- Extracts proxy ID
- Calls `simple.Render()` + `InvokeEvent("UpdateEvent",())`
- Returns: `{filterId: int, proxyId: int, success: bool, error?: string}`

**`visualization.filters.delete`** (FILT-01.4):
- Params: `{filterId: int}` (the proxy ID from the create response)
- Calls `simple.Delete(filterObj)` where filterObj is retrieved from the proxy registry by filterId
- Calls `simple.Render()` + `InvokeEvent("UpdateEvent",())`
- Returns: `{success: bool, error?: string}`

**`visualization.filters.list`** (FILT-01.5):
- No params
- Returns: `{filters: [{id: int, type: "clip"|"contour"|"streamtracer", parameters: object}]}`

Key implementation notes:
- Use `simple.GetActiveSource()` as the input for each filter
- Each filter proxy ID should be unique integer (use Python's `id()` or a counter)
- Track created filters in a class-level dict: `_filters: dict[int, {"type": str, "proxy": object}]`
- Call `simple.Render()` after every state-changing operation
- Invoke `"UpdateEvent"` to trigger viewport push to connected WebSocket clients
- Wrap all filter operations in try/except, return `{"success": False, "error": str(e)}` on failure
- Do NOT call `simple.Render()` before confirming the filter was created successfully (avoid race with viewport push)
