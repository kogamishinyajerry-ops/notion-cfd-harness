<introduction>

## The Core Problem

The current `paraview_web_launcher.py` launches the container with:

```
--entrypoint pvpython openfoam/openfoam10-paraview510 lib/site-packages/vtkmodules/web/launcher.py /tmp/launcher_config.json
```

`--entrypoint pvpython` replaces the image entrypoint entirely with the `pvpython` binary, which then runs `launcher.py`. There is no hook to inject Python code before `launcher.py` starts the wslink server. Custom `@exportRpc` decorated classes in `adv_protocols.py` would only register if Python imports them before `launcher.py` initializes -- but with the current docker run command, no such import happens.

## The Solution

Two-part approach:

**Build-time (Dockerfile):** Create a custom Docker image `cfd-workbench:openfoam-v10` based on `openfoam/openfoam10-paraview510` that bakes in an entrypoint wrapper script (`/entrypoint_wrapper.sh`). The wrapper imports `adv_protocols.py` (if present at `/tmp/adv_protocols.py`) and then execs the original `pvpython lib/site-packages/vtkmodules/web/launcher.py` command.

**Runtime (paraview_web_launcher.py):** Update `_start_container()` to:
1. Mount `adv_protocols.py` from the host at `/tmp/adv_protocols.py:ro` inside the container
2. Use `--entrypoint /entrypoint_wrapper.sh` instead of `--entrypoint pvpython`
3. Pass through `pvpython lib/site-packages/vtkmodules/web/launcher.py /tmp/launcher_config.json` as CMD args

**Import ordering guaranteed:** The wrapper script runs first (as PID 1), imports `adv_protocols.py`, then execs `pvpython`. The exec replaces the shell process with pvpython, so wslink only starts after the import completes.

  <name>Task 2: Write the entrypoint_wrapper.sh script</name>
  <files>entrypoint_wrapper.sh</files>
  <read_first></read_first>
  <action>
Create `/Users/Zhuanz/Desktop/notion-cfd-harness/entrypoint_wrapper.sh` -- the script that becomes PID 1 inside every ParaView Web session container. This script:

1. **Imports `adv_protocols.py`** (mounted at `/tmp/adv_protocols.py`) using `pvpython -c "import sys; sys.path.insert(0, '/tmp'); import adv_protocols"`. This registers any `@exportRpc` decorated classes with wslink's global registry BEFORE the launcher starts.
2. **On import error:** Prints error to stderr prefixed with `ADV_PROTOCOL_ERROR:`, then exits with code 1. Container exits immediately. `_wait_for_ready()` detects the container crash and surfaces the error message in the Dashboard viewer area.
3. **On success:** Executes the original pvpython command (everything passed as CMD args, e.g. `pvpython lib/site-packages/vtkmodules/web/launcher.py /tmp/launcher_config.json`) using `exec "$@"` so pvpython replaces PID 1.

```sh
#!/bin/bash
# entrypoint_wrapper.sh — PID 1 inside ParaView Web session containers.
# Imports adv_protocols.py (if present) before starting the wslink server.

set -euo pipefail

PROTOCOLS_FILE="/tmp/adv_protocols.py"

# Attempt to import adv_protocols.py using pvpython.
# This registers @exportRpc decorated classes with wslink's global registry.
if [ -f "$PROTOCOLS_FILE" ]; then
    echo "[entrypoint_wrapper] Found $PROTOCOLS_FILE, importing protocols..."
    # Use pvpython to import so ParaView/Python paths are set up correctly.
    # The import must succeed (rc=0) before we proceed.
    if ! pvpython -c "import sys; sys.path.insert(0, '/tmp'); import adv_protocols; print('adv_protocols imported OK')"; then
        echo "[entrypoint_wrapper] ADV_PROTOCOL_ERROR: failed to import $PROTOCOLS_FILE" >&2
        exit 1
    fi
    echo "[entrypoint_wrapper] Protocols registered with wslink."
else
    echo "[entrypoint_wrapper] $PROTOCOLS_FILE not found — skipping protocol import."
fi

# Exec the original command (e.g., pvpython launcher.py ...).
# exec replaces this shell with pvpython, so wslink starts after import.
exec "$@"
```

Make the file executable: `chmod +x entrypoint_wrapper.sh`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Host filesystem -> Container | adv_protocols.py mounted read-only into container; case_dir mounted read-only |
| Docker daemon | Launcher runs as unprivileged user inside container; --platform linux/amd64 enforced |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-01 | Tampering | entrypoint_wrapper.sh modified on host | mitigate | File is inside project repo; Dockerfile COPY command copies it into image at build time |
| T-19-02 | Denial of Service | adv_protocols.py import hangs forever | mitigate | entrypoint_wrapper.sh uses `set -e`; container exits on pvpython non-zero exit; `_wait_for_ready()` detects container crash |
| T-19-03 | Information Disclosure | Mounted adv_protocols.py exposes host path | accept | File is project code, not secrets; mounted read-only |
| T-19-04 | Elevation of Privilege | Container escapes to host | accept | Default Docker isolation; not project scope to harden container security |
</threat_model>
