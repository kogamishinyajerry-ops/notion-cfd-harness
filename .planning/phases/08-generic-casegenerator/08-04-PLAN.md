**The Problem:**
- `_body_in_channel_vertices()` generates vertices as a 4-column x 4-row grid with body interior skipped
- `_generate_blockmesh_blocks()` for BODY_IN_CHANNEL currently returns `[]` (empty)
- This produces invalid blockMeshDict with `blocks ( )` and 0 hex blocks

**The Fix:**
Replace the empty return with proper 8-block hex generation.

For a 4-column (nx=4) x 4-row (ny=4) vertex grid with a 2x2 body hole in the center:
- The outer ring consists of 8 hex blocks: 4 on the left of body, 4 on the right of body
- Each block maps 2D cell region to 2 hexes (z=0 + z=thickness layers)

The vertex grid produces vertices in row-major order (y then x within each z-layer):
- For z=0 layer: row0_all_x, row1_left+body+right, row2_left+body+right, row3_all_x

  - Left columns (x0, x1 at y1, y2): 2 columns × 2 rows × 1 z = 4 vertices

  - Total z=thickness layer: 16 vertices (v16-v31)
