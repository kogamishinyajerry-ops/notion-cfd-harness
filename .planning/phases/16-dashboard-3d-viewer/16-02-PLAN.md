# Key data flow
1. job.result stores auto-launched ParaView Web session info (from Phase 15 job_service.py):
   job.result.paraview_session_id = "PVW-{hex}"
   job.result.paraview_session_url = "ws://localhost:{port}/ws"
   job.result.output_dir = "/path/to/openfoam/case"

2. When viewer tab activates:
   - If job.result.paraview_session_id exists: reuse existing session
   - Otherwise: launch new session via POST /api/v1/visualization/launch with caseDir = job.result.output_dir

3. Pass to ParaViewViewer:
   jobId = job.id
   caseDir = job.result.output_dir
</context>
