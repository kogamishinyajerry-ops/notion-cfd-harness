## Copywriting Contract

| Element | Copy | When Shown |
|---------|------|------------|
| Primary CTA | `Launch 3D Viewer` | On completed jobs with a case directory |
| Viewer loading heading | `Initializing viewer` | While WebSocket connects + ParaView Web starts |
| Viewer loading body | `Connecting to ParaView Web session...` | First 3s (within timeout) |
| Viewer loading spinner | CSS animation (no text) | After heading shown |
| Error: connection refused | `Cannot connect to ParaView Web. Ensure Docker is running and port {port} is available.` | WebSocket connection refused |
| Error: auth failed | `Authentication failed. The session may have expired. Refresh to try again.` | WebSocket auth rejected |
| Error: session not found | `Session not found. The viewer session may have timed out (30 min idle). Refresh to launch a new session.` | 404 on session status check |
| Disconnected banner | `Connection lost. Reconnecting...` | WebSocket drops, auto-reconnect in progress |
| Reconnect exhausted | `Viewer disconnected after {n} attempts.` | After max retries exceeded |
| Empty state (no completed job) | `No completed job. 3D visualization is available for completed jobs only.` | Job status != completed |
| Session heartbeat | `POST /api/v1/visualization/{session_id}/activity` every 60s | While viewer is active |

**Destructive actions:** None in Phase 16. Session auto-shutdown is backend-managed (30-min idle timeout). No user-facing shutdown button in this phase.

## Component Inventory

### ParaViewViewer.tsx

**Purpose:** React component that renders a ParaView Web viewport using the protocol.js client library.

**Props:**
```typescript
interface ParaViewViewerProps {
  sessionId: string;       // PVW-{8-char-hex}
  sessionUrl: string;      // ws://localhost:{port}/ws
  authKey: string;         // Secrets.token_urlsafe(16)
  jobId: string;
  onError?: (reason: string) => void;
  onConnected?: () => void;
}
```

**States:**

| State | Visual | Trigger |
|-------|--------|---------|
| `idle` | Empty container with "Launch 3D Viewer" button | Component mounted, no session launched |
| `launching` | Button disabled, spinner visible | Launch requested |
| `connecting` | "Initializing viewer" skeleton/pulse | WebSocket opening |
| `connected` | Full 3D canvas (ParaView Web renders here) | WebSocket open + authenticated |
| `disconnected` | Banner "Connection lost. Reconnecting...", auto-retry | WebSocket close event |
| `reconnect-exhausted` | Banner with retry button | >5 reconnect attempts failed |
| `error` | Error card with message + retry option | API error or auth failure |

**Reconnection behavior:**
- Backoff: 1s, 2s, 4s, 8s, 16s (exponential, max 5 attempts)
- On reconnect success: reset attempt count, resume session
- On max attempts exceeded: show `reconnect-exhausted` state

**Heartbeat:** `POST /api/v1/visualization/{session_id}/activity` every 60 seconds while connected.

**File location:** `dashboard/src/components/ParaViewViewer.tsx`
**CSS file:** `dashboard/src/components/ParaViewViewer.css`

### Integration into JobDetailPage

- Add `viewer` tab to `activeTab` union: `'logs' | 'output' | 'config' | 'viewer'`
- Tab visible only when `job.status === 'completed'` (Phase 16 scope)
- On tab activation: call `POST /api/v1/visualization/launch` with `{job_id, case_dir}` to get session credentials
- Pass `session_id`, `session_url`, `auth_key` to `<ParaViewViewer />`
- On tab deactivation: no action (session persists for 30 min idle)
- API client method to add:
  ```typescript
  async launchVisualizationSession(jobId: string, caseDir: string): Promise<VisualizationLaunchResponse>
  async getVisualizationSession(sessionId: string): Promise<VisualizationStatusResponse>
  ```

**New files to create:**
- `dashboard/src/components/ParaViewViewer.tsx`
- `dashboard/src/components/ParaViewViewer.css`
- `dashboard/src/services/paraview.ts` (API client methods for visualization endpoints)
